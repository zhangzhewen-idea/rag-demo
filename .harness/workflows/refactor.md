# 重构工作流

## 上下文加载

必读 `constitution.md`、`security-boundaries.md`、`project-context.md`、`architecture.md`、`commands.md` 和 `quality-gates.md`，再读取重构目标、现有调用路径、测试和公开契约。

## 执行步骤

1. 检查 Git 状态，定义必须保持不变的外部行为、接口、数据和性能边界。
2. 运行受影响范围的基线测试；基线失败时先区分既有问题，不能把失败带入重构完成声明。
3. 制定可回滚的小步方案，优先保持编译和测试持续可用。
4. 按 COLA 依赖方向移动职责，避免机械造层、空抽象、同层入口互调和跨层捷径。
5. 每个关键步骤运行最小测试，完成后运行受影响模块完整门禁和 ArchUnit。
6. 比较重构前后公开行为，审查最终 diff，展示提交摘要，只提交重构文件。

## 退出条件

- 外部行为和契约保持不变，除非用户明确要求同步改变。
- 代码职责和依赖方向更清晰，没有新增无价值抽象。
- 基线对应测试、完整门禁和 diff 检查通过，并已创建范围准确的中文 commit。
- 无法证明行为等价、需要扩大公开变更或门禁受阻时不提交，报告风险和证据。
