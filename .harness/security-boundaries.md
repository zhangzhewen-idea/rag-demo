# 安全边界

## 可自主执行

- 读取、搜索、创建和修改项目内代码、测试及中文文档。
- 安装本地依赖，运行格式化、测试、构建和只读诊断命令。
- 在当前任务范围内修复由自身改动引起的问题。
- 验证通过后，仅暂存当前任务文件并创建简洁中文 commit。

## 必须获得用户明确批准

- 删除文件、目录、业务数据或 Git 历史。
- 修改 `.env`、密钥、Token、证书、真实凭据或 CI/CD 配置。
- 执行 `git push`、`git rebase`、`git reset --hard`、强制推送或其他历史改写。
- 生产部署、公开发布、破坏性数据库迁移或影响共享环境的操作。
- 执行会产生费用、发送问题/证据/答案或影响共享数据的 Chat、Embedding、Rerank、Judge 和管理员评估运行。
- 删除或重建 `rag-demo-index-v2`，以及修改已执行 Flyway migration 或执行 `flyway repair`。

授权必须针对具体操作；允许 commit 不代表允许 push，允许本地启动不代表允许生产部署。目标或范围无法明确解析时先做只读检查，再请求确认。

## 密钥与数据

- 不读取、输出或提交真实密钥、私有文档内容、数据库转储或用户数据。
- 数据库密码、Redis 密码、JWT 密钥和百炼 API Key 只通过环境变量注入。
- 不创建或修改 `.env` 来绕过缺失配置；需要凭据时报告所需变量名，由用户在本地提供。
- 日志、异常、测试夹具和最终回复不得包含密码、Token、API Key 或完整私有文档内容。
- `SimpleLoggerAdvisor` 的 DEBUG 日志可能包含问题、会话历史和知识库证据；排障只读取必要片段，不复制完整私有内容，
  非调试场景使用 `AI_INTERACTION_LOG_LEVEL=INFO`。
- 检查配置时只确认变量名、默认值风险和注入路径，不回显当前环境变量值。

## 外部副作用与文件安全

- 普通验证优先使用单元测试、fake/mock 和本地构建，避免真实模型、共享数据库和共享 Redis 副作用。
- 启动检查前确认命令边界并设置有界请求；不会自动清理、迁移或重建共享数据。
- 上传文件只能保存在配置的受控根目录，路径必须规范化并防止越界。
- 删除文档、向量、索引或业务记录属于数据删除；即使应用提供相应接口，也必须先获得批准。
- Flyway migration 一旦在共享或本地既有数据库执行，不做格式化或重排；checksum 不一致先只读比对，不直接 repair。
- 暂存使用明确路径，提交前核对 `git diff --cached`，避免混入无关文件和用户改动。
